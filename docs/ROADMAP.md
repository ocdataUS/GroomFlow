# ROADMAP — GroomFlow Delivery

| Sprint | Focus | Key Outcomes |
| --- | --- | --- |
| **S0 — Enhanced Bootstrap** | Plugin shell, assets, Elementor stub | Admin menu, capability scaffolding, REST namespace, placeholder Kanban preview. |
| **S1 — Data Foundations & Admin CRUD** | Custom tables + management screens | CRUD for clients, guardians, services, packages, flags, views; settings for thresholds/polling; seed defaults. |
| **S2 — REST & Business Logic** | Intake + workflow APIs | Visit intake, stage move logic, history logging, public tokenized boards, CLI scaffolds. |
| **S3 — Kanban UX & Modal Editing** | Interactive board experience | Drag/drop with timers/capacity alerts, modal editor with inline edits, polling engine, lobby mode. |
| **S4 — Elementor & Shortcode Polish** | Deep customization tooling | Elementor controls (style + metadata ordering), view switcher, shortcode parity, public/private display configuration. |
| **S5 — Notifications & Reporting** | Communication + insights | Stage-triggered notifications, KPI dashboard, CSV/PDF exports, accessibility/perf polish, automated smoke QA. |

Re-evaluate scope at the end of every sprint. Update this roadmap and sprint docs whenever priorities shift. Document all decisions in breadcrumbs.
